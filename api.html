CrawlScript API

ECMA

CrawlScript是在ECMA标准的基础上设计的语言，是一种javascript,在语法上和网页开发中使用的javascript完全相同，但是并不支持网页开发中所特有的函数和对象，如document、alert()在CrawlScript中会失效。CrawlScript有自己的一套特有的函数和对象，详细请看下面的API。

$

$是CrawlScript最核心的一个对象，它封装了爬虫常用一些方法，为了简化操作，$也可以当一个函数使用。

$(url)

根据网页的url获取一个网页的DOM元素集合。


实例-显示网页的所有文字:
doc=$("http://www.baidu.com");
print(doc);

$.each(elements,selector,func)

在一个DOM元素集合elements中根据selector选择出满足条件的元素，将每个元素作为func的第一个参数，执行func函数。func函数形式为function(element,i)，i为遍历的顺序标号。


实例-显示网页中所有超链接的链接:
doc = $("http://www.baidu.com");
func = function( e ){
  print(e.attr( "href" ));
}
$.each(doc , "a" , func);

$.extract(settings)

对一个网页的指定元素进行指定操作，一般用于信息抽取和爬虫链接生成。
settings是一个JSON对象，JSON需要3个参数 url、selector、func,url是要解析的网页的url，selector要处理的DOM元素的选择器，func是一个函数，表示对每个被选择的元素要执行的函数，原型为function(element,i),element是被选择的元素，i是遍历的顺序标号。

DOM对象集合

DOM对象集合最初都是由$(url)函数返回的，执行doc=$(url),doc为整个网页DOM元素的集合，执行doc.div()、doc.ul(3).li(),doc.select(selector)等函数，可以根据条件获得doc的子集。详细用法请看下面的API，下文中用doc代表一个已经生成的DOM对象集合。

doc.select(selector)

根据选择器selector返回doc中符合选择器条件的DOM元素集合。

selector是一个字符串，对DOM元素进行条件限制，例如"div>p"表示DOM元素是div里的p元素，"a[id=link]"表示DOM元素是id=link的a元素，CrawlScript中的selector和Jsoup中的selector完全等价，详细请看selector说明书。


实例-显示网页中所有超链接的链接的文本:
doc = $("http://www.baidu.com");
print(doc.select("a"));

doc.div() doc.div(index)

doc.div()返回doc中所有的div元素组成的集合,doc.div(index)表示由doc中第index个div元素组成的集合。

还有一些类似的函数，下面API中不详细说明。doc.a(),doc.a(index),doc.ul(),doc.ul(index)......

由于doc.div()返回的还是DOM元素集合，所以此类函数经常连续着使用，如：doc.div().p().a()

doc.div() doc.div(index)

doc.div()返回doc中所有的div元素组成的集合,doc.div(index)表示由doc中第index个div元素组成的集合。

还有一些类似的函数，下面API中不详细说明。doc.a(),doc.a(index),doc.ul(),doc.ul(index)......

由于doc.div()返回的还是DOM元素集合，所以此类函数经常连续着使用，如：doc.div().p().a()
